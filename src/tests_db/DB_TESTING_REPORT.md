# Отчёт по автоматизированному тестированию базы данных

## Цель тестирования

После обновления структуры базы данных и сервисного слоя приложения CareForMe было проведено повторное автоматизированное тестирование бизнес-логики проекта.

Целями тестирования являлись:

* проверка корректности работы сервисного слоя;
* проверка обработки пользовательских действий;
* проверка работы системы растений, достижений, уровней и советов;
* выявление возможных ошибок после изменений базы данных и программного кода;
* подтверждение работоспособности приложения после интеграции изменений из ветки second_version_frontend.



## Используемые инструменты

* Python 3.10
* pytest
* unittest.mock
* pytest-cov



## Протестированные сервисы

### UserService

Проверены:

* получение статистики пользователя;
* получение текущего уровня;
* работа ежедневной серии входов;
* получение информации об уровнях;
* работа слотов растений;
* разблокировка дизайнов;
* смена горшков и леек;
* обновление пользовательской статистики.

Количество тестов: 28



### FlowerService

Проверены:

* посадка растений;
* полив растений;
* обработка ошибок пользователя;
* изменение освещения;
* изменение местоположения растения;
* проверка здоровья растения;
* рост растения;
* достижение зрелости;
* восстановление погибшего растения.

Количество тестов: 34



### ChallengeService

Проверены:

* получение достижений;
* расчёт прогресса достижений;
* регистрация ошибок пользователя;
* регистрация идеального выращивания;
* регистрация гибели растения;
* получение статистики достижений.

Количество тестов: 15



### LevelQuestService

Проверены:

* получение заданий уровня;
* получение прогресса пользователя;
* выполнение заданий;
* проверка специальных условий заданий;
* повышение уровня;
* выдача наград;
* получение статуса всех уровней;
* обработка максимального уровня пользователя.

Тесты были адаптированы под актуальную игровую механику ветки **second_version_frontend**, включая обновлённую систему уровней.

Количество тестов: 22



### TipsService

Проверены:

* получение всех советов;
* получение позитивных советов;
* поиск советов по типу;
* получение советов для растений;
* обработка ошибок парсинга данных.

Количество тестов: 11
